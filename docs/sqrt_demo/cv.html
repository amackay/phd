Syntax Definition cv 37
Description: This syntax construction states that a variable 𝑥, which has been declared to be a setvar variable by $f statement vx, is also a class expression. This can be justified informally as follows. We know that the class builder {𝑦 ∣ 𝑦 ∈ 𝑥} is a class by cab http://us.metamath.org/mpeuni/cab.html . Since (when 𝑦 is distinct from 𝑥) we have 𝑥 = {𝑦 ∣ 𝑦 ∈ 𝑥} by cvjust http://us.metamath.org/mpeuni/cvjust.html , we can argue that the syntax "class 𝑥 " can be viewed as an abbreviation for "class {𝑦 ∣ 𝑦 ∈ 𝑥}". See the discussion under the definition of class in [Jech] p. 4 showing that "Every set can be considered to be a class".

While it is tempting and perhaps occasionally useful to view cv 37 as a "type conversion" from a setvar variable to a class variable, keep in mind that cv 37 is intrinsically no different from any other class-building syntax such as cab http://us.metamath.org/mpeuni/cab.html 37 , cun http://us.metamath.org/mpeuni/cun.html 37 , or c0 http://us.metamath.org/mpeuni/c0.html 37 .

For a general discussion of the theory of classes and the role of cv 37, see mmset.html#class 37.

(The description above applies to set theory, not predicate calculus. The purpose of introducing class 𝑥 here, and not in set theory where it belongs, is to allow to express, i.e., "prove", the weq http://us.metamath.org/mpeuni/weq.html 37 of predicate calculus from the wceq 38 of set theory, so that we do not overload the = connective with two syntax definitions. This is done to prevent ambiguity that would complicate some Metamath parsers.)
